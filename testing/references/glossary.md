# Glossary

Concise definitions for common testing terms.

- **Assertion**: a statement that checks whether actual behavior matches expectation.
- **Code coverage**: how much code runs while tests execute.
- **Library**: reusable functions, usually shipped as a package.
- **Testing framework**: a set of testing-oriented libraries and runtime behavior for running and organizing tests.
- **Stub**: a replacement that returns predefined values.
- **Spy**: a function wrapper that records calls.
- **Mock**: a double that can both replace behavior and verify interactions.
- **Test double**: general term covering stubs, spies, and mocks.
- **Monkey patching**: swapping object properties or functions at test time.
- **Test case**: a single scenario with setup, action, and expectation.
- **Test suite**: a group of related test cases.
- **Assertion**: the concrete check that decides whether a test passes or fails.
- **Matcher**: a reusable comparison rule used by an assertion.
- **Setup / cleanup hooks**: logic that prepares or resets state around tests.
- **Suite-level hooks**: setup or cleanup that runs once for a whole group of tests.
